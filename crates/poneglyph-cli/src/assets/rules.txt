## poneglyph: durable memory + code graph

This project has poneglyph wired up (MCP server `poneglyph mcp`). Prefer
its tools over re-deriving things or manually scanning directories:

- `remember` / `recall` / `get_project_context` — durable cross-session
  memory. Call `get_project_context` at session start, `recall` before
  re-researching something, `remember` for durable facts/decisions/preferences.
- `codegraph_explore` (`callers_of:`/`callees_of:`/`imports_of:`/`tests_for:`/
  `path:<a>..<b>`, or a bare keyword for a graph-backed name search) — one call
  returns verbatim source + call path + blast radius. ALWAYS try this FIRST
  for "find X" / "what calls/imports/breaks if I change X" questions. Trust
  codegraph's results; don't re-verify with grep. Fall back to grep/glob
  only when the graph has nothing. Requires `poneglyph graph init` once.
